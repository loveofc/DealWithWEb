데이터를 그룹으로 처리할 수 있는 방법은 배열을 이용하는 것.

문제점 2가지:
크기를 변경할 수 없다
하나의 배열에 다른 타입의 데이터는 저장할 수 없다.

Collection 객체를 이용하면 이러한 제한사항을 벗어나 데이터를 그룹으로 처리할 수 있다
collection 객체는 java.util 패키지 안에 있다. 객체 이름은 set, list, map의 단어가 포함되어 있다.

collection 객체 가운데 Enumeration과 Iterator가 있습니다. 이 두 객체의 공통점은 그룹 안에 있는 요소에
접근할 때 인덱스나 키로 접근하는 것이 아니고, 커서 개념으로 접근한다는 점입니다.
주로 collection 안에 있는 모든 요소를 차례로 접근할 때 사용하면 편리하고 속도 면에서 빠르다.

실제 개발하다보면 하나의 collection 안에 아무런 상관관계가 없는 객체들이 저장되는 경우가 별로 없고, 
같은 타입의 객체들이 한 곳에 저장되는 경우가 많다.
collection 객체에서 자료를 추출할때 타입 캐스팅이 번거롭다 그래서 탄생한게 제네릭이다.

ArrayList<String> list = new ArrayList<String>();
타입을 미리 정해두는 문법이다.
